## Registry coverage conclusions

- All 21 canonical module definitions are represented above.
- Route-less modules are not rendered as pinned navigation targets.
- Entitlements remain canonical in `lib/platform/entitlements/service.ts`.
- Install and pin state remain distinct; unpinning never removes module data.
- A browser-verified designation is intentionally withheld until authenticated flows run at the required viewports.
